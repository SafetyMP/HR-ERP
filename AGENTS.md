# Agent instructions — HR ERP

This repository follows the **HR Product Owner operating model**. The canonical workflow is packaged as the Cursor project skill [`.cursor/skills/hr-product-owner/`](.cursor/skills/hr-product-owner/SKILL.md); load it when scoping features, writing UAC, or judging UX friction.

Before writing or reviewing application code for a new capability, read:

- [HR Product Owner operating model](docs/product/hr-product-owner-operating-model.md)
- Active Feature briefs live in [docs/product/feature-briefs/](docs/product/feature-briefs/)
- New capabilities start from [Feature brief template](docs/product/feature-brief-template.md)
- **Compliance / wage & hour rule packs:** [docs/compliance/](docs/compliance/) (jurisdiction matrix, invariants, retention, acceptance criteria for backend agents). Agents should load the project skill [`.cursor/skills/hr-backend-compliance/SKILL.md`](.cursor/skills/hr-backend-compliance/SKILL.md) (`hr-backend-compliance`) when implementing or reviewing pay/time/compliance logic.
- **Payroll & compensation calculation kernel** (deterministic gross-to-net, proration, versioned tables): project skill [`.cursor/skills/hr-payroll-calculation-engine/SKILL.md`](.cursor/skills/hr-payroll-calculation-engine/SKILL.md) (`hr-payroll-calculation-engine`). Implementation lives under [`packages/payroll-calc/`](packages/payroll-calc/). The repo **Orchestrator** ([`.cursor/rules/orchestrator.mdc`](.cursor/rules/orchestrator.mdc)) runs this skill **conditionally** in sequencing step 3 and expects it on delegated **Implementation** / **QA** Tasks whenever executable pay math, fingerprints/replay fields, or that package surface—see orchestrator bullets **Payroll calculation kernel** and Cursor **Task** instructions.

- **Architecture governance** ([docs/architecture/README.md](docs/architecture/README.md)) — bounded contexts, module proposals, SLO/load gates, contracts (REST/gRPC). Agents load project skill [`.cursor/skills/hr-erp-principal-architecture/SKILL.md`](.cursor/skills/hr-erp-principal-architecture/SKILL.md) (**`@hr-erp-principal-architecture`**) when designing schemas, service boundaries, Kafka/async contracts, or cross-context APIs (Orchestrator: attach on Architecture Tasks per [`.cursor/rules/orchestrator.mdc`](.cursor/rules/orchestrator.mdc)).
- **Database migrations & state custody** — zero-downtime posture, per-database migrations, RLS-safe seeds/loaders, post-migrate verification. Agents load project skill [`.cursor/skills/hr-db-migration-state/SKILL.md`](.cursor/skills/hr-db-migration-state/SKILL.md) (**`@hr-db-migration-state`**) when authoring or reviewing `prisma/schema.prisma`, `prisma/migrations/**`, `services/*/db/migrations/**`, data backfills, dual-write cutovers, or extending [`scripts/db-verify-migration.ts`](scripts/db-verify-migration.ts). Canonical runbook: [docs/architecture/database-migrations-and-state.md](docs/architecture/database-migrations-and-state.md). For reuse outside this repo, copy or symlink to `~/.cursor/skills/hr-db-migration-state/`.

## Orchestration

Sequence is defined in [`.cursor/rules/orchestrator.mdc`](.cursor/rules/orchestrator.mdc). **Order:** **step 1 — `hr-product-owner` PO orchestration checkpoint** (Feature brief or spike ADR; emit checkpoint before Architecture) → **PO context pinned** in downstream artifacts → **Architecture** ( **`@hr-erp-principal-architecture`** per orchestrator on Architecture Tasks) → **`hr-erp-innovation-rd` Innovation / R&D parity** (mandatory; `@hr-erp-innovation-rd` or [.cursor/skills/hr-erp-innovation-rd/SKILL.md](.cursor/skills/hr-erp-innovation-rd/SKILL.md)) → Legal → **`hr-backend-compliance`** (conditional when pay/time/earnings/compliance paths or [`docs/compliance/`](docs/compliance/) change; `@hr-backend-compliance` or [.cursor/skills/hr-backend-compliance/SKILL.md](.cursor/skills/hr-backend-compliance/SKILL.md)) → **`hr-payroll-calculation-engine`** (conditional when gross-to-net / `packages/payroll-calc` / fingerprint replay / `computePayroll` surfaces change; `@hr-payroll-calculation-engine` or [.cursor/skills/hr-payroll-calculation-engine/SKILL.md](.cursor/skills/hr-payroll-calculation-engine/SKILL.md); use **with** `hr-backend-compliance` when both apply) → **`hr-ai-data-governance`** (conditional when ML screening, scoring, governance APIs, or `docs/ai-governance/` / `lib/governance/` change; `@hr-ai-data-governance` + **agent-ai-governance** on delegated Tasks) → Integration (if needed) → Implementation → **`hr-code-health` mandatory pass** (`@hr-code-health`) → **`hr-erp-security-identity` mandatory Security / identity pass** (`@hr-erp-security-identity` + **agent-security**) → Security review → QA (**`hr-erp-qa-chaos`** + **agent-qa** on QA Tasks and test-stack Implementation Tasks unless orchestrator **QA chaos N/A** verbatim). Populate the **Innovation / R&D parity** and (when triggered) **AI data governance parity** tables in [`specs/templates/architecture-spec.md`](specs/templates/architecture-spec.md) unless the orchestrator documents chore **N/A** lines. Delegated **Task** agents on non-trivial features must attach **`hr-product-owner`** + brief/ADR path through the pipeline (Orchestrator chore **N/A** only); delegated **Task** agents touching payroll/time must attach `hr-backend-compliance` and **agent-legal-hr-compliance** when matrices or compliance packs apply, and **`hr-payroll-calculation-engine`** when executable pay-math or kernel paths apply; delegated Tasks on AI scoring/screening/governance flows must attach **`hr-ai-data-governance`** and **agent-ai-governance** when orchestrator triggers apply; delegated **Implementation** Tasks must attach `hr-code-health` + **agent-code-health** unless the prompt carries orchestrator **Code health N/A** verbatim, and **`hr-erp-security-identity`** + **agent-security** on the same Tasks (plus prisma/middleware/security-lib Tasks per orchestrator) unless **Security skill N/A** verbatim; delegated **QA** Tasks (and Implementation Tasks that materially change `tests/**`, `lib/qa/**`, Vitest/Playwright QA CI, or [`docs/QA.md`](docs/QA.md)) must attach **`hr-erp-qa-chaos`** + **agent-qa** unless **QA chaos N/A** verbatim.

**Rules of engagement**

- **Orchestration:** Non-trivial features follow [`.cursor/rules/orchestrator.mdc`](.cursor/rules/orchestrator.mdc): step 1 runs **`hr-product-owner`**, verifies the brief or spike ADR, and records a **PO orchestration checkpoint** before Architecture. Delegated Tasks must include the same skill + brief path through the pipeline.
- **Engineering:** Do not start build work for a capability until a Feature brief exists and the PO gate items (user, pain, outcome, scope boundary) are filled.
- **QA:** Execute tests against the numbered UAC in the Feature brief verbatim; unclear UAC is a product defect, not an assumption for QA. For chaos-style suites, synthetic mega-fixtures, temporal concurrency, and mandatory **`FAILURE_SUMMARY`** handoffs, load project skill [`.cursor/skills/hr-erp-qa-chaos/SKILL.md`](.cursor/skills/hr-erp-qa-chaos/SKILL.md) (`hr-erp-qa-chaos`) and [docs/QA.md](docs/QA.md).
- **Frontend:** Primary flows must not require a manual; meet the friction checks in the brief (including task-time targets where specified).
- **FinOps / multi-agent token efficiency (Cursor Tasks):** When **two or more** delegated **Task** subagents run on one feature thread, the Orchestrator loads [`.cursor/skills/hr-erp-finops-swarm/SKILL.md`](.cursor/skills/hr-erp-finops-swarm/SKILL.md) (`hr-erp-finops-swarm`) per [`.cursor/rules/orchestrator.mdc`](.cursor/rules/orchestrator.mdc) (**Transcript fidelity** bullet **7**): model-tier defaults, **FinOps note**, and **ping-pong circuit breaker**. Optional rule: [`.cursor/rules/agent-finops.mdc`](.cursor/rules/agent-finops.mdc) (`agent-finops`) for explicit Human cost gates. **Not** a substitute for in-app inference budgets—those stay under [SLM-first inference ADR](docs/architecture/adrs/0001-slm-first-inference-routing.md) and **`hr-erp-mlops`**.
- **Collaboration / swarm post-mortem (Principal Systems Auditor):** project skill [`.cursor/skills/hr-erp-collaboration-audit/SKILL.md`](.cursor/skills/hr-erp-collaboration-audit/SKILL.md) (`hr-erp-collaboration-audit`). Invoke `@hr-erp-collaboration-audit` or attach when the Human Lead requests choreography retrospectives, swarm integrity scoring, transcript audits, golden-thread leakage analysis, or “how did the agents collaborate”—the Orchestrator references **bullet __8__** in [`.cursor/rules/orchestrator.mdc`](.cursor/rules/orchestrator.mdc). Templates: [`specs/templates/golden-thread-trace-table.md`](specs/templates/golden-thread-trace-table.md), [`specs/templates/golden-thread-regulated-payroll-drill.md`](specs/templates/golden-thread-regulated-payroll-drill.md), [`specs/templates/swarm-multiagent-rerun-checklist.md`](specs/templates/swarm-multiagent-rerun-checklist.md). Example write-up: [docs/meta/collaboration-post-mortem-transcript-49235615.md](docs/meta/collaboration-post-mortem-transcript-49235615.md). For reuse outside this repo, copy or symlink to `~/.cursor/skills/hr-erp-collaboration-audit/`.

The Product Owner role owns empathy mapping, prioritization, UAC, and friction standards — not implementation code.
